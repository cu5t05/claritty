# AI Use Policy

## Training Prohibition

The code, documentation, configuration, and all other contents of this repository may not be used to train, fine-tune, or otherwise develop AI or machine learning models without explicit written permission from the licensor.

This prohibition applies to:

- The source code in all files
- Documentation including README, GETTING_STARTED, and all markdown files
- Configuration structures and formats
- Comments, commit messages, and issue or discussion content

This restriction is asserted in addition to the terms of the Apache License 2.0 with Common Clause. Where this policy and the license overlap, the more restrictive interpretation applies.

---

## AI-Generated Contributions

Claritty does not accept AI-generated contributions. This is a philosophical position, not a technical one.

Claritty's architecture is built on the principle that AI proposes and humans decide. Accepting AI-generated code as a contribution would contradict that principle at the most fundamental level — the codebase itself.

Contributors affirm that their submissions are their own original work by posting the required rights declaration in the Discussion thread before any code is considered. AI-generated code cannot satisfy that declaration.

---

## AI Tooling in Development

Claritty is developed without AI code generation tools. GitHub Copilot is disabled for this repository. This is consistent with the project's philosophy and is a visible, deliberate policy statement.

---

## Permitted AI Use

Using Claritty itself to interact with AI models is the intended use case. The system is designed for controlled, consent-gated human-AI collaboration. Running Claritty with an AI provider is fully permitted and encouraged.
